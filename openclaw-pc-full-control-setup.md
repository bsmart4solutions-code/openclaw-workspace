## 第三步：一次性设置完整远程权限（核心步骤）

> ⚠️ 这是整个流程里**唯一需要手动执行**的步骤。
> 完成后，云端助手可以自动执行一切，不再需要你在电脑前。

复制以下命令，在 WSL2 终端**一次性执行**：

```bash
# 添加所有需要的命令白名单
openclaw approvals allowlist add --agent "*" "/home/$USER/.npm-global/bin/openclaw" && \
openclaw approvals allowlist add --agent "*" "/usr/bin/bash" && \
openclaw approvals allowlist add --agent "*" "/bin/bash" && \
openclaw approvals allowlist add --agent "*" "/usr/bin/node" && \
openclaw approvals allowlist add --agent "*" "/usr/bin/npm" && \
openclaw approvals allowlist add --agent "*" "/usr/bin/git" && \
openclaw approvals allowlist add --agent "*" "/usr/bin/curl" && \
openclaw approvals allowlist add --agent "*" "/usr/bin/wget" && \
openclaw approvals allowlist add --agent "*" "/bin/sh" && \
openclaw approvals allowlist add --agent "*" "/mnt/c/Program Files/Google/Chrome/Application/chrome.exe" && \
openclaw approvals allowlist add --agent "*" "/mnt/c/Windows/System32/cmd.exe" && \
openclaw approvals allowlist add --agent "*" "/usr/bin/powershell.exe" && \
openclaw approvals set-default --agent "*" allow
```

验证白名单：
```bash
openclaw approvals list
```

应该看到所有命令已列出，且 Default 为 `allow`。

## 第四步：让 OpenClaw Gateway 开机自启

```bash
# 设置 systemd 用户服务开机自启
systemctl --user enable openclaw-gateway
systemctl --user start openclaw-gateway

# 确认运行中
systemctl --user status openclaw-gateway
```

> 💡 WSL2 默认不会随 Windows 启动自动运行 systemd 服务。  
> 建议配合第五步的开机脚本使用。

## 第五步：设置 Windows 开机自动启动 WSL2 + OpenClaw

在 Windows 端创建一个启动脚本，让电脑开机后自动运行 OpenClaw。

### 方法 A：Task Scheduler（推荐）

1. 按 `Win + R`，输入 `taskschd.msc`，打开任务计划程序
2. 点击右侧「创建基本任务」
3. 名称：`OpenClaw AutoStart`
4. 触发器：选「计算机启动时」
5. 操作：选「启动程序」
6. 程序：`C:\Windows\System32\wsl.exe`
7. 参数：`-d Ubuntu -e bash -c "systemctl --user start openclaw-gateway"`
8. 勾选「无论用户是否登录都要运行」
9. 完成

### 方法 B：启动文件夹脚本

在 `C:\Users\你的用户名\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\` 创建 `start-openclaw.bat`：

```bat
@echo off
wsl -d Ubuntu -e bash -c "systemctl --user start openclaw-gateway"
```

## 🛠️ 常见问题

**Q: 电脑重启后云端断连了？**  
A: 检查第四、五步是否正确设置了开机自启。

**Q: 命令执行被拒绝（approval timed out）？**  
A: 检查 `openclaw approvals list`，确认目标命令在白名单里，或 Default 为 `allow`。

**Q: 如何查看远程执行日志？**  
A: `journalctl --user -u openclaw-gateway -f`

**Q: 如何断开远程控制？**  
A: `openclaw gateway disconnect` 或 `systemctl --user stop openclaw-gateway`

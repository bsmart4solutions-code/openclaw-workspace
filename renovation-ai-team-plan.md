## 2. 8 个 Agent 详细规格

### 🤖 Agent 1: 市场研究 Agent (Market Research Agent)

**职责：** 持续监控和分析马来西亚装修市场

| 项目 | 详情 |
|------|------|
| **核心功能** | 行业趋势分析、竞争对手监控、市场洞察 |
| **工作内容** | 1. 每周生成市场报告<br>2. 追踪高端装修趋势<br>3. 分析华人/马来客户偏好<br>4. 监控竞争对手动态 |
| **输入来源** | Google Search、Facebook 竞品页面、行业报告 |
| **输出** | 周报（Markdown/PDF） |
| **执行频率** | 每周一次 |
| **所需工具** | gsk search, gsk crawl, browser |

---

### 🤖 Agent 2: 内容创作 Agent (Content Creation Agent)

**职责：** 运营 Facebook 专页，持续产出高质量内容

| 项目 | 详情 |
|------|------|
| **核心功能** | 帖文创作、排程发布、内容策略 |
| **工作内容** | 1. 根据市场趋势创作内容<br>2. 制作高质量图片/视频<br>3. 排程发布时间<br>4. A/B 测试不同内容风格 |
| **内容类型** | - 装修案例展示（Before/After）<br>- 设计灵感<br>- 装修知识（避坑指南）<br>- 客户见证<br>- 促销活动 |
| **目标平台** | Facebook, Instagram |
| **执行频率** | 每天 1-2 篇 |
| **所需工具** | gsk image_generation, gsk video_generation, browser (发布) |

---

### 🤖 Agent 3: 客服 Agent (Customer Service Agent)

**职责：** 第一时间响应客户咨询，筛选潜在客户

| 项目 | 详情 |
|------|------|
| **核心功能** | 自动回复、初步筛选、预约安排 |
| **工作内容** | 1. 监控 FB/IG/WhatsApp 消息<br>2. 自动回复常见问题<br>3. 收集客户需求（面积、预算、风格）<br>4. 初步筛选并打标签<br>5. 预约现场测量时间 |
| **常见问题库** | - 价格范围<br>- 工期多久<br>- 保修政策<br>- 设计风格<br>- 案例参考 |
| **响应时间目标** | < 5 分钟 |
| **所需工具** | message tool (Telegram/WhatsApp), browser |

---

### 🤖 Agent 4: Lead 跟进 Agent (Lead Nurturing Agent)

**职责：** 培育潜在客户，推动转化

| 项目 | 详情 |
|------|------|
| **核心功能** | 跟进漏斗管理、转化推动 |
| **工作内容** | 1. 跟进未成交的潜在客户<br>2. 发送跟进信息（3天/7天/14天）<br>3. 发送案例和报价<br>4. 推动预约再次沟通<br>5. 记录跟进历史 |
| **客户分层** | - A 类：高意向（已报价）<br>- B 类：中意向（了解中）<br>- C 类：低意向（观望） |
| **转化目标** | B → A: 30%, A → 成交: 40% |
| **所需工具** | CRM (Notion/Airtable), message tool |

---

### 🤖 Agent 5: 项目管理 Agent (Project Management Agent)

**职责：** 追踪装修项目进度，协调资源

| 项目 | 详情 |
|------|------|
| **核心功能** | 进度追踪、任务提醒、节点管理 |
| **工作内容** | 1. 记录每个项目的关键节点<br>2. 自动提醒下一步任务<br>3. 回答客户"进度到哪里了"<br>4. 协调设计师/承包商<br>5. 预警延期风险 |
| **项目阶段** | 咨询 → 测量 → 设计 → 报价 → 签约 → 施工 → 验收 → 完工 |
| **提醒规则** | - 每个阶段开始/结束前 1 天提醒<br>- 延期风险提前 3 天预警 |
| **所需工具** | Notion/Airtable, calendar, message tool |

---

### 🤖 Agent 6: 财务报价 Agent (Quotation & Finance Agent)

**职责：** 生成准确报价，追踪付款

| 项目 | 详情 |
|------|------|
| **核心功能** | 报价生成、成本计算、财务追踪 |
| **工作内容** | 1. 根据客户需求生成报价单<br>2. 计算材料成本 + 人工 + 利润<br>3. 追踪付款状态<br>4. 发送付款提醒<br>5. 记录每个项目的利润 |
| **报价要素** | - 项目明细<br>- 材料规格<br>- 工期<br>- 付款条款<br>- 保修期 |
| **准确率目标** | > 95% |
| **所需工具** | Spreadsheet/Notion, PDF generator |

---

### 🤖 Agent 7: 口碑管理 Agent (Reputation Agent)

**职责：** 管理在线评价，提升品牌形象

| 项目 | 详情 |
|------|------|
| **核心功能** | 评价监控、回应管理、口碑建设 |
| **工作内容** | 1. 监控 Google Business 和 FB 评价<br>2. 及时回应好评（感谢）<br>3. 处理差评（道歉 + 解决方案）<br>4. 请求客户给好评<br>5. 收集客户见证 (Testimonials) |
| **评分目标** | Google: 4.8+, FB: 4.5+ |
| **差评响应时间** | < 24 小时 |
| **所需工具** | Google Alerts, browser (监控 FB/Google) |

---

### 🤖 Agent 8: 竞品分析 Agent (Competitive Intelligence Agent)

**职责：** 持续监控竞争对手，找出差异化机会

| 项目 | 详情 |
|------|------|
| **核心功能** | 竞品监控、策略分析、机会识别 |
| **工作内容** | 1. 监控 5-10 个主要竞争对手<br>2. 分析他们的价格、服务、内容<br>3. 追踪他们的促销活动<br>4. 找出我方差异化优势<br>5. 提出改进建议 |
| **监控维度** | - 价格<br>- 服务范围<br>- 客户评价<br>- 内容策略<br>- 设计风格 |
| **报告频率** | 双周一次 |
| **所需工具** | gsk search, browser, social monitoring |

## 3. 技术实现方案

### 3.1 架构设计

```
                    ┌─────────────────────┐
                    │    OpenClaw Gateway │
                    │    (18789)          │
                    └─────────┬───────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        │                     │                     │
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│ Telegram Bot  │    │  WhatsApp     │    │  Facebook     │
│ (管理后台)     │    │  Business     │    │  Page         │
└───────────────┘    └───────────────┘    └───────────────┘
        │                     │                     │
        └─────────────────────┼─────────────────────┘
                              │
                    ┌─────────┴─────────┐
                    │   数据存储层       │
                    │  - Notion         │
                    │  - Airtable       │
                    │  - SQLite         │
                    │  - Google Sheet   │
                    └───────────────────┘
```

### 3.2 实现方式

| 功能 | 实现方式 |
|------|----------|
| **召唤 Agent** | `sessions_spawn` (OpenClaw Sub-agents) |
| **消息处理** | Telegram Bot API / WhatsApp Business API |
| **数据存储** | Notion API / Airtable |
| **内容生成** | GSK (图片、视频、文字) |
| **定时任务** | `openclaw cron` |
| **浏览器操作** | `browser` tool (自动发布 FB) |

### 3.3 Agent 通讯流程

```
用户发送消息
    │
    ▼
Telegram/WhatsApp/FB
    │
    ▼
OpenClaw Gateway
    │
    ▼
中央协调 Agent (判断任务类型)
    │
    ├──▶ 市场研究 Agent ──▶ 生成报告 ──▶ 发送给你
    │
    ├──▶ 客服 Agent ──▶ 回复客户 ──▶ 更新 CRM
    │
    ├──▶ Lead 跟进 Agent ──▶ 发送跟进 ──▶ 记录结果
    │
    └──▶ 项目管理 Agent ──▶ 检查进度 ──▶ 提醒相关方
```

## 7. 风险与解决方案

| 风险 | 影响 | 解决方案 |
|------|------|----------|
| **AI 回复不够人性化** | 客户体验下降 | 设置人工复核机制，复杂问题转人工 |
| **报价不准确** | 利润损失或丢单 | 人工复核报价，定期更新成本数据 |
| **FB 算法变化** | 内容曝光下降 | 保持多元化渠道，不要完全依赖 FB |
| **数据丢失** | 业务中断 | 定期备份，重要数据本地 + 云端双存储 |
| **客户隐私泄露** | 法律风险 | 数据加密，限制 Agent 访问敏感信息 |

# Session Insights (会话洞察)

从学习与会话历史中浮现模式。

## 触发

说 "show stats"、"how am I doing"、"analytics"、"insights"、"heatmap" 或 "correction rate" 时。

## 数据来源

计算指标前从这些位置收集数据：

```bash
# 会话历史和学习
cat .claude/LEARNED.md 2>/dev/null || cat CLAUDE.md | grep -A999 "LEARNED"
cat .claude/learning-log.md 2>/dev/null

# 会话活动
git log --oneline --since="today" --author="$(git config user.name)"
git diff --stat
```

**纠正** 是用户重定向、修复或覆盖代理输出的任何实例。计算 `[LEARN]` 条目和会话历史中的显式纠正标记。

## 显示内容

### 会话总结

```
会话洞察
  时长: 47 分钟
  改动: 23 个文件修改
  纠正: 2 次自我纠正
  学习: 3 个新模式捕获
  上下文: 62% 使用 (安全)
```

### 学习分析

```
学习洞察 (42 总计)

最高分类:
  测试     12 学习 (29%)
  导航     8 学习 (19%)
  Git      7 学习 (17%)
  质量     6 学习 (14%)

最多应用:
  #12 [测试] 提交前运行测试 — 15 次
  #8  [导航] 确认常用名称的路径 — 11 次

过时学习 (从未应用):
  #15 [编辑] 偏好命名导出 — 0 次 (45 天旧)
```

### 纠正热力图

```
纠正热力图

按分类 (全部时间):
  ████████████ 测试      34 次纠正
  ████████     导航     22 次纠正
  ██████       Git      18 次纠正
  ████         质量      12 次纠正

热学习 (最多纠正最少学习):
  - [测试] Mock 外部依赖 — 纠正 8x, 学习 0x
    → 考虑: /learn-rule 永久捕获

冷学习 (学习但从未应用):
  - [编辑] 使用命名导出 — 45 天前学习, 应用 0x
    → 如果不再相关考虑删除
```

## 原则

- 按相关性排序，不是按时间
- 包含原始错误上下文让学习可操作
- 标记高纠正率需要额外注意的区域
- 如果没有匹配学习，明确说明

## 输出

- 会话统计 (时长、文件、纠正、学习)
- 学习分析 (分类分布、应用频率)
- 纠正热力图 (按分类、热点/冷点)
- 建议改进领域

# Learn Rule (学习规则)

从错误中捕捉教训，存入持久化记忆。

## 触发场景

用户说：
- "记住这个"
- "添加到规则"
- "别再那样做了"
- "记下来"
- "从这学到一个教训"

或者你发现自己犯了错误后。

## 工作流程

1. **识别教训** — 犯了什么错？应该怎么做？
2. **格式化规则** — 带上完整上下文
3. **提出建议** — 等用户批准
4. **持久化** — 存入 LEARNED 区或项目记忆

## 格式

```
[LEARN] 分类: 一句话规则
错误: 哪里出了问题
修正: 应该如何纠正
```

## 分类

| 分类 | 示例 |
|------|------|
| Navigation | 文件路径、找代码、改错文件 |
| Editing | 代码改动、模式、错误方法 |
| Testing | 测试方法、覆盖缺口、不稳定测试 |
| Git | 提交、分支、合并问题 |
| Quality | Lint、类型、风格违规 |
| Context | 何时需要澄清、缺少需求 |
| Architecture | 设计决策、错误的抽象 |
| Performance | 优化、O(n^2) 循环、内存 |

## 示例

```
最近错误: 修改了错误的 utils.ts 文件

[LEARN] Navigation: 多个文件同名时要确认完整路径。

添加到 LEARNED 区？(y/n)
```

## 原则

- 一定要等用户批准后再持久化
- 规则保持在一行内 — 具体且可操作
- 差: "写好代码" → 好: "数据库列必须用 snake_case"
- 包含错误上下文，让规则之后还有意义

## 输出

- 提议的 [LEARN] 规则（带分类）
- 持久化后的确认

# Cost Tracker (成本追踪)

监控和优化 AI 会话成本。

## 触发场景

- 检查会话成本
- 设置预算警报
- 优化 token 支出
- 规划多会话工作

## 成本意识

### 检查当前成本
```bash
# 会话成本在每个会话结束时显示
# 中途：检查状态栏或运行 /cost
```

### 成本驱动因素

| 操作 | 相对成本 | 优化 |
|------|----------|------|
| 大文件读取 | 高 | 使用 offset/limit 参数 |
| 广泛 grep 搜索 | 中 | 限定特定目录 |
| 生成子代理 | 高（新上下文） | 通过 SendMessage 重用代理 |
| 重复工具调用 | 累积 | 批量操作 |
| MCP 工具调用 | 变化 | 最小化往返 |
| 模型选择 | 3-10x 差异 | 简单任务用 haiku |

### 按任务类型的 Token 预算

| 任务 | 典型成本 | 预算警报 |
|------|----------|----------|
| Bug 修复 | $0.10-0.50 | $1.00 |
| 功能（小） | $0.50-2.00 | $3.00 |
| 功能（大） | $2.00-8.00 | $10.00 |
| 重构 | $1.00-5.00 | $7.00 |
| 代码审查 | $0.20-1.00 | $2.00 |

### 优化策略

1. **缩小提示范围** — "Fix the auth bug in src/auth/login.ts" vs "Fix the auth bug"
2. **用对的模型** — 简单查询用 haiku，功能用 Sonnet，架构用 Opus
3. **委托给子代理** — 搜索/探索操作在子代理中保持主上下文精简
4. **主动压缩** — 不要等自动压缩；任务边界时压缩
5. **选择性读取** — 大文件用 `offset` 和 `limit` 参数
6. **批量操作** — 一条消息中多个独立工具调用

## 预算警报

设置心理检查点：
- **50% 预算** — 进度正常吗？应该压缩吗？
- **80% 预算** — 结束当前任务，避免新探索
- **100% 预算** — 提交已有的，开始新会话

## 输出

```text
成本追踪
  会话成本: $X.XX
  Token 使用: [输入]K / [输出]K
  缓存命中率: ~XX%

  最高成本驱动:
    1. [操作] — $X.XX
    2. [操作] — $X.XX

  优化建议:
    - [具体建议]
```

## 规则

- 在开始复杂任务前检查成本基线
- 大改动设置预算上限
- 如果接近预算，优先完成当前任务而不是探索新问题
